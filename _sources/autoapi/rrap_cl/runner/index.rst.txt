.. py:function:: evaluate(rme_files_path: str, nsims: int, deploy_model_fn: str, prod_model_fn: str, lm_model_fn: str, results_dir: str, metrics: list = None, uncertainty_dict: dict = None, active_models: set = None, nprocs: int = 1, costs_only: bool = False, sample_scale: bool = False, distance_override_NM: float = None, coral_only: bool = False, seed: int = None, intervention_year_offset: int = 1) -> list[str]

   Evaluate costs of intervention scenarios.

   :param rme_files_path: Path to ReefMod Engine results.
   :type rme_files_path: str
   :param nsims: Number of simulations to evaluate.
   :type nsims: int
   :param deploy_model_fn: Path to deployment spreadsheet model, including filename but excluding file extension.
   :type deploy_model_fn: str
   :param prod_model_fn: Path to production spreadsheet model, including filename but excluding file extension.
   :type prod_model_fn: str
   :param results_dir: Path to directory for storing results.
   :type results_dir: str
   :param lm_model_fn: Path to LM spreadsheet model, including filename but excluding file extension.
   :type lm_model_fn: str
   :param metrics: List of metrics to calculate. Default is None.
   :type metrics: list, optional
   :param uncertainty_dict: Dictionary specifying uncertainty parameters. Default is None.
   :type uncertainty_dict: dict, optional
   :param active_models: Intervention types to include in cost calculations.  Valid values are
                         ``"outplant"`` and ``"lm"``.  Defaults to both when ``None``.
                         Pass ``{"outplant"}`` for CA-only or ``{"lm"}`` for LM-only scenarios.
   :type active_models: set, optional
   :param nprocs: Number of parallel worker processes for cost sampling.  Each worker
                  receives ``ceil(nsims / nprocs)`` draws.  Defaults to 1 (serial).
   :type nprocs: int, optional
   :param costs_only: When ``True``, skips post-processing of ecological metric files.
                      Use this when only cost outputs are needed (e.g. in
                      ``run_cost_exploration``).  Defaults to ``False``.
   :type costs_only: bool, optional
   :param distance_override_NM: When set, replaces the geographic port-distance calculation with this
                                fixed value (nautical miles) for all reefsets.  Useful for best-guess
                                explorer runs where the config best-point distance (e.g. 27 NM North,
                                54 NM Centre) should be used instead of the computed reef distance.
   :type distance_override_NM: float, optional
   :param coral_only: If True, only use coral-related metrics (RCI_3 and RFI), excluding COTS and Rubble.
   :type coral_only: bool, default=False
   :param seed: Random seed for reproducibility.
   :type seed: int, optional
   :param intervention_year_offset: Number of years to subtract from the RME intervention year when labelling
                                    costs in outputs.  ReefMod Engine records the year corals first become
                                    *detectable*; the actual production and deployment work takes place one
                                    year earlier.  The default of 1 therefore produces cost outputs whose
                                    year columns reflect the calendar year costs were actually incurred.
                                    Pass 0 to keep the raw RME year unchanged.
   :type intervention_year_offset: int, default=1

   :returns: Paths to result files.
   :rtype: list[str]

.. py:function:: run_cost_exploration(rme_template_path: str, nsims: int, deploy_model_fn: str, prod_model_fn: str, lm_model_fn: str, results_dir: str, assessment_year: int, reefset_CA: list = None, reefset_LM: list = None, metrics: list = None, uncertainty_dict: dict = None, nprocs=1, intervention_year_offset: int = 1) -> dict

   Explore cost uncertainty across intervention scenarios.

   Prepares a scenario directory from the RME template (filtering to year 1
   through ``assessment_year`` and optionally updating reef assignments), then
   calls ``evaluate()`` three times — combined (CA + LM), CA-only, and
   LM-only, writing each run's cost outputs into a labelled subdirectory
   under ``results_dir``.  Ecological metric post-processing is skipped as
   only cost outputs are of interest here.

   :param rme_template_path: Path to the RME output directory to use as a template.
   :type rme_template_path: str
   :param nsims: Number of Monte Carlo draws for cost uncertainty sampling.
   :type nsims: int
   :param deploy_model_fn: Path to deployment spreadsheet model (excluding extension).
   :type deploy_model_fn: str
   :param prod_model_fn: Path to production spreadsheet model (excluding extension).
   :type prod_model_fn: str
   :param lm_model_fn: Path to LM spreadsheet model (excluding extension).
   :type lm_model_fn: str
   :param results_dir: Root directory for outputs.  Three subdirectories are created:
                       ``combined/``, ``ca_only/``, and ``lm_only/``.
   :type results_dir: str
   :param assessment_year: The year corals from the second deployment first become detectable
                           in RME (i.e. the RME detection year, not the cost year).  Year 1
                           (first deployment) is always included; rows beyond ``assessment_year``
                           are dropped from the scenario before running.  Cost outputs will be
                           labelled ``assessment_year - intervention_year_offset`` (default: one
                           year earlier than this value).
   :type assessment_year: int
   :param intervention_year_offset: Passed through to ``evaluate()``.  See that function's docstring for
                                    full details.  The default of 1 subtracts one year from RME detection
                                    years so that cost outputs reflect the calendar year in which
                                    production and deployment actually occurred.
   :type intervention_year_offset: int, default=1
   :param reefset_CA: Reef IDs to assign to ``reefset_CA`` (e.g. ``["18-096"]``).
                      If ``None``, the template value from ``scenario_info.json`` is used.
   :type reefset_CA: list, optional
   :param reefset_LM: Reef IDs to assign to ``reefset_LM`` (e.g. ``["16-071"]``).
                      If ``None``, the template value from ``scenario_info.json`` is used.
   :type reefset_LM: list, optional
   :param metrics: Passed to ``evaluate()`` for ID key generation.  Ecological metric
                   files are not post-processed.  Defaults to ``[rci, raw_rti, rfi]``.
   :type metrics: list, optional
   :param uncertainty_dict: Uncertainty parameters for ecological metrics. Defaults to
                            ``default_uncertainty_dict()``.
   :type uncertainty_dict: dict, optional
   :param nprocs: Number of cores to use.
   :type nprocs: int, optional

   :returns: Mapping of scenario label to list of result file paths:
             ``{"combined": [...], "ca_only": [...], "lm_only": [...]}``.
   :rtype: dict


.. py:function:: summarise_mc_results(results_dir: str, quantiles: list = None, scenario_id: int = 1) -> dict

   Summarise Monte Carlo cost exploration results produced by ``run_cost_exploration``.

   Reads the combined cost overview CSV from each scenario subdirectory
   (``combined/``, ``ca_only/``, ``lm_only/``), computes per-year quantiles
   across draws for each cost column, and returns a nested dict suitable for
   JSON serialisation and HTML visualisation.

   :param results_dir: Root directory passed to ``run_cost_exploration`` (parent of the
                       ``combined/``, ``ca_only/``, and ``lm_only/`` subdirectories).
   :type results_dir: str
   :param quantiles: Quantile levels to compute, e.g. ``[0.05, 0.25, 0.5, 0.75, 0.95]``.
                     Defaults to ``[0.05, 0.25, 0.5, 0.75, 0.95]``.
   :type quantiles: list, optional
   :param scenario_id: Scenario ID used in the cost overview filename.  Defaults to ``1``.
   :type scenario_id: int, optional

   :returns: Structure: ``{scenario: {cost_col: {year: {quantile_label: value}}}}``.
             Also written to ``<results_dir>/mc_summary.json``.
   :rtype: dict

.. py:function:: evaluate_lm_cost(workbook_path: str, scenarios_df: pandas.DataFrame, **factors) -> pandas.DataFrame

   Evaluate the LM cost model over a year-by-year coral scenario and apply
   the inventory/replacement model to produce a cost schedule.

   For each year in ``scenarios_df``, the required pool count is derived as
   ``clip(ceil(number of corals / yield_per_pool), range_lower, range_upper)``
   using the bounds from the LM config. Capex/opex are read back from the
   spreadsheet each year, and the inventory model is applied across the full
   time series.

   :param workbook_path: Absolute path to the LM Excel workbook including extension (.xlsx).
   :type workbook_path: str
   :param scenarios_df: Year-by-year coral scenario table with columns ``year`` and
                        ``number of corals``. Typically a single rep/intervention slice of
                        ``iv_yearly_scenarios.csv``. Will be sorted by year internally.
   :type scenarios_df: pd.DataFrame
   :param \*\*factors: Factor name-value pairs to override from config best-point values.
                       Do not pass ``larval release pools`` — it is derived from
                       ``scenarios_df`` each year.

   :returns: ``scenarios_df`` (sorted by year) extended with columns: ``capex``,
             ``opex``, ``inventory``, ``new_capex_scale``,
             ``new_capex_replacement``, ``total_capex``, ``total_capex_opex``,
             ``ratio``, ``average``.
   :rtype: pd.DataFrame


.. py:function:: run_cost_model(workbook_path: str, params_df: pandas.DataFrame, *, model_type: str | None = None, nprocs: int | None = None) -> pandas.DataFrame

   Evaluate a cost model over a set of parameter combinations.

   The model type (production or deployment) and config version are inferred
   from the workbook filename — e.g. ``'3.9.1 CA Production Model.xlsx'``.
   Pass ``model_type`` explicitly only when the filename does not follow the
   standard naming convention.

   When ``nprocs`` is greater than 1, ``params_df`` is split into chunks and
   evaluated in parallel — each worker opens its own temporary workbook copy.
   Otherwise the whole DataFrame is evaluated serially in a single workbook
   session.

   Columns not present in ``params_df`` default to the values currently in the
   workbook.

   :param workbook_path: Absolute path to the Excel workbook including extension (.xlsx).
   :type workbook_path: str
   :param params_df: Each row is one model evaluation. Column names must be factor names from
                     the relevant config CSV. Only factors to be overridden from workbook
                     defaults need to be included.
   :type params_df: pd.DataFrame
   :param model_type: ``"production"`` or ``"deployment"``. Inferred from the filename if
                      not provided.
   :type model_type: str, optional
   :param nprocs: Number of parallel worker processes. Values <= 1 run serially.
                  Defaults to serial (``None``).
   :type nprocs: int, optional

   :returns: Input DataFrame with three appended columns: ``capex``, ``opex``,
             and ``total_cost``.
   :rtype: pd.DataFrame


.. py:function:: sample_joint_factors(nsims: int, seed=None) -> tuple[pandas.DataFrame, pandas.DataFrame, pandas.DataFrame, SALib.ProblemSpec]

   Generate Sobol' samples over the combined factor space of all three models
   (production, deployment, and LM), then slice back into per-model DataFrames.

   Building one combined ProblemSpec ensures every row corresponds to the same
   sample point and that any shared factors are identical across models by
   construction.

   Categorical flooring and discrete value mapping are intentionally *not*
   applied here — ``run_cost_model`` handles those transformations to avoid
   double-application.

   :param nsims: Desired total number of model evaluations. The base Sobol' sample count
                 N is derived via ``get_NK(nsims, n_combined_factors)``.
   :type nsims: int
   :param seed: Random seed for reproducibility.
   :type seed: int or None

   :returns: * **prod_samples** (*pd.DataFrame*) -- Raw Sobol' samples for the production model.
             * **deploy_samples** (*pd.DataFrame*) -- Raw Sobol' samples for the deployment model.
             * **lm_samples** (*pd.DataFrame*) -- Raw Sobol' samples for the LM model.
             * **combined_sp** (*ProblemSpec*) -- The combined ProblemSpec used for sampling.

.. py:function:: sweep_ca(prod_model: str, deploy_model: str, sweep_param: str, search_range, prod_params: dict = None, dep_params: dict = None) -> pandas.DataFrame

   Sweep a single parameter across the CA production and deployment models.

   :param prod_model: Path to production cost model workbook.
   :type prod_model: str
   :param deploy_model: Path to deployment cost model workbook.
   :type deploy_model: str
   :param sweep_param: Name of the parameter to sweep over.
   :type sweep_param: str
   :param search_range: Values to sweep over.
   :type search_range: iterable
   :param prod_params: Fixed factor overrides for the production model.
   :type prod_params: dict, optional
   :param dep_params: Fixed factor overrides for the deployment model. Any key also present
                      in ``prod_params`` is overwritten by the production value, keeping
                      shared factors (e.g. ``num_1yoec``, ``coral_yield_1YOEC``) consistent.
   :type dep_params: dict, optional

   :returns: One row per sweep value with columns: ``search_range``,
             ``prod_capex``, ``prod_opex``, ``dep_capex``, ``dep_opex``,
             ``total_cost``.
   :rtype: pd.DataFrame
